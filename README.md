# readingtracker-python
由于最近的读书需求，我使用python给自己制作了一个读书追踪系统
# 读书打卡系统 V3｜单文件傻瓜式桌面版

本项目是一个本地运行的读书计划与打卡工具。V3 版的目标是：尽量不使用命令行，通过一个桌面窗口完成书单导入、阅读计划生成、每日打卡、历史记录修改、日历查看、统计报告、系统建议、备份恢复和邮件提醒。

日常使用时，主要打开：

```text
reading_app_v3.py
```

或双击：

```text
run_gui_v3.bat
```

---

## 一、V3 版升级内容

V3 版相较前面的版本，主要做了这些改进：

1. 整合为单一 GUI 主程序；
2. 支持打包为 exe；
3. 增加首次使用向导；
4. 支持简洁打卡和深度打卡；
5. 支持推迟一天、跳过本书、暂停本书、临时加书；
6. 支持历史打卡记录的查看、修改和删除；
7. 增加月历视图；
8. 增加一键备份和恢复；
9. 优化错误提示；
10. 增加系统建议，包括完成率、阅读速度、计划强度、拖延情况和阅读类别平衡。

## 二、推荐项目结构

建议将文件放在同一个项目文件夹中，例如：

```text
D:\HJX_505\
├── reading_app_v3.py              # 主程序，日常主要运行这个
├── run_gui_v3.bat                 # 双击启动脚本
├── build_exe_v3.bat               # 一键打包 exe 脚本
├── requirements_v3.txt            # 依赖列表
├── README.md                      # 使用说明
├── data\
│   ├── books.csv                  # 书单
│   ├── config.json                # 配置
│   ├── reading_plan.csv           # 阅读计划
│   ├── reading_log.csv            # 打卡记录
│   ├── reading_notes.csv          # 阅读笔记
│   └── reading_summary.csv        # 每周复盘
├── reports\                      # 阅读报告
├── figures\                      # 统计图
```

程序会自动创建 `data`、`reports`、`figures`、`backup` 文件夹。

## 三、环境配置

### 1. 安装 Python

建议使用 Python 3.10 或以上版本。

安装完成后，在 PowerShell 中输入：

```powershell
python --version
```

如果能显示版本号，例如：

```text
Python 3.11.8
```

说明 Python 安装正常。

---

### 2. 进入项目文件夹

假设你的项目路径是：

```text
D:\HJX_505
```

在 PowerShell 中输入：

```powershell
cd D:\HJX_505
```

---

### 3. 创建虚拟环境

```powershell
python -m venv .venv
```

---

### 4. 激活虚拟环境

```powershell
.venv\Scripts\activate
```

激活成功后，命令行前会出现：

```text
(.venv)
```

---

### 5. 安装依赖

V3 版主要依赖 Python 标准库。生成图表和打包 exe 时需要额外依赖。

推荐安装：

```powershell
pip install -r requirements_v3.txt
```

如果没有 `requirements_v3.txt`，可以手动安装：

```powershell
pip install matplotlib pyinstaller
```

其中：

```text
matplotlib：用于生成统计图
pyinstaller：用于打包 exe
```

## 六、阅读强度说明

| 阅读强度 | 大致含义 | 适合情况 |
|---|---:|---|
| 轻量 | 每天约25分钟 | 忙碌时期，保持习惯 |
| 标准 | 每天约45分钟 | 长期稳定阅读 |
| 高强度 | 每天约75分钟 | 假期或集中阅读 |
| 冲刺 | 每天约120分钟 | 短期冲刺，不建议长期使用 |

程序会根据书籍难度自动调整每日页数。难度越高，每天分配页数越少。

## 七、书单 CSV 格式

书单 CSV 建议包含以下列：

```csv
书名,作者,总页数,已读页数,类别,难度,优先级,是否必读,状态,备注
```

示例：

```csv
书名,作者,总页数,已读页数,类别,难度,优先级,是否必读,状态,备注
《万物简史》,Bill Bryson,544,0,科学通识,1,1,是,未开始,从自然科学整体图景开始
《人类简史》,Yuval Noah Harari,440,0,历史与社会思想,1,2,是,未开始,训练宏大叙事的理解与怀疑
《思考，快与慢》,Daniel Kahneman,512,0,认知心理与决策,3,3,是,未开始,理解判断偏差和启发式
```

字段说明：

| 字段 | 是否必填 | 说明 |
|---|---|---|
| 书名 | 必填 | 书籍名称 |
| 作者 | 建议填写 | 作者名称 |
| 总页数 | 必填 | 必须是大于0的数字 |
| 已读页数 | 建议填写 | 没读过填0 |
| 类别 | 建议填写 | 如哲学、经济学、金融学、统计、运动科学 |
| 难度 | 建议填写 | 1—5 |
| 优先级 | 建议填写 | 数字越小越先读 |
| 是否必读 | 建议填写 | 是/否 |
| 状态 | 可选 | 未开始、阅读中、暂停、已完成、跳过 |
| 备注 | 可选 | 阅读目标、版本说明等 |

难度建议：

```text
1 = 很轻松
2 = 通俗读物
3 = 有一定概念密度
4 = 专业通识或经典导读
5 = 半教材或高密度理论书
```

## 十一、日历页

日历页用于查看月度任务安排。

颜色含义：


## 十二、打卡页

打卡页用于记录实际阅读情况。

### 1. 基础流程

1. 选择打卡日期；
2. 点击“加载任务”；
3. 选择当天任务；
4. 选择打卡模式；
5. 填写实际页码和阅读分钟；
6. 点击“保存打卡”或“保存并重排”。

推荐点击：

```text
保存并重排
```

这样程序会根据真实阅读进度自动调整后续计划。

---

### 2. 简洁打卡

适合时间少、只想快速记录时使用。

主要填写：

```text
实际起始页
实际结束页
阅读分钟
完成状态
一句话总结
```

---

### 3. 深度打卡

适合精读或训练批判性思维时使用。

除基础字段外，还需要填写：

```text
核心概念
作者观点
论证链
关键证据
隐含前提
可反驳之处
可迁移观点
心理学或法学连接
今日问题
```

如果当天较忙，建议至少填写：

```text
核心概念
作者观点
可反驳之处
今日问题
```

## 十六、邮件提醒设置

邮件提醒的作用是：每天早上向邮箱发送今日阅读任务。

### 1. 设置 SMTP 授权码

PowerShell 临时设置：

```powershell
$env:READING_SMTP_PASSWORD="你的SMTP授权码"
```

永久设置：

```powershell
setx READING_SMTP_PASSWORD "你的SMTP授权码"
```

永久设置后，需要重新打开 PowerShell 或重新启动程序。

注意：多数邮箱使用的是 SMTP 授权码，不是邮箱登录密码。

---

### 2. 配置邮件参数

进入：

```text
设置与备份
```

填写：

```text
是否启用：是
收件邮箱：你的邮箱
发件邮箱：你的邮箱
SMTP服务器：smtp.qq.com / smtp.163.com / smtp.gmail.com
SMTP端口：465 或 587
SMTP密码环境变量：READING_SMTP_PASSWORD
```

---

### 3. 测试邮件

点击：

```text
预览邮件
```

确认内容正常后，再点击：

```text
发送测试邮件
```

---

### 4. 创建每天8点提醒任务

点击：

```text
创建8点提醒任务
```

程序会在 Windows 中创建任务计划：

```text

```

前提条件：

```text
1. 电脑早上8点开机
2. 网络正常
3. 邮箱SMTP服务已开启
4. SMTP授权码环境变量已设置
```

## 二十、常见问题

### 1. 为什么计划记录数比计划天数多？

因为某些日期可能同时包含：

```text
阅读任务
每周复盘任务
```

所以90天计划可能生成100条以上记录，这是正常现象。

---

### 2. 为什么导入书单失败？

通常是 CSV 缺少必要字段。

请确认至少包含：

```text
书名
总页数
```

并且总页数是大于0的数字。

---

### 3. 为什么生成图表失败？

需要安装：

```powershell
pip install matplotlib
```

如果使用 exe 版，需要在打包前安装 matplotlib，并重新打包。

---

### 4. 为什么邮件发送失败？

检查：

```text
1. 是否启用邮件提醒；
2. 收件邮箱、发件邮箱是否正确；
3. SMTP服务器和端口是否正确；
4. 邮箱是否开启SMTP服务；
5. 是否使用SMTP授权码，而不是邮箱登录密码；
6. 环境变量是否设置成功；
7. 是否重新启动程序。
```

---

### 5. 为什么历史记录修改后计划没有变化？

修改历史记录后，需要点击：

```text
重新计算并重排
```

或者在打卡时使用：

```text
保存并重排
```

---

### 6. 为什么书被暂停后还出现在计划里？

暂停或跳过书籍后，需要重新生成或重排计划。

可以点击：

```text
首页 → 生成/更新计划
```

或：

```text
设置与备份 → 保存并重排计划
```
